# 実装時のコメント規約

## 1. ファイル冒頭コメント
各ファイルの先頭に、このファイルの責務を2〜5行で説明するコメントを追加してください。
以下の観点を含めること:
- このファイルが全体フローのどこを担当するか
- 主な入出力
- 特に重要な副作用や設計上の注意点

## 2. 関数コメント
各主要関数の直前に、必要に応じて以下の観点を含むコメントを追加してください。
- この関数の役割
- 何を受け取り、何を返すか
- state を更新する場合は何を更新するか
- fallbackや例外吸収を行う場合はその理由

## 3. 分岐コメント
複雑な if / match / conditional edge の直前には、
「何を判定しているか」ではなく「なぜここで判定が必要か」を説明するコメントを追加してください。

## 4. マジックナンバー・閾値コメント
閾値、confidence、timeout、budget関連の数値を使っている箇所には、
その数値自体の意味と、何の判断に使われるかをコメントしてください。
